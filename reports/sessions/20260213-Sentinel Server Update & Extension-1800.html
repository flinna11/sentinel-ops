üèóÔ∏è Stage 1: The "Crawl" (Skeleton Framework)

Goal: Establish physical connectivity, basic virtualization, and your primary communication hub.

Phase I: Infrastructure Reset & IPs

  • Wipe Metadata: Clean old Longhorn and RabbitMQ data from your 500GB SSDs to ensure a fresh start.

  • Static IPs: Use your router's DHCP reservation to lock the IPs for Optiplex 1, Optiplex 2, and the Raspberry Pi.

  • Connectivity: Ping all devices from your Windows 11 PC to confirm they are reachable.

Phase II: The Golden Template (Lite)

  • Build VM: Create a single Ubuntu 24.04 VM.

  • Install Basics: Install curl, qemu-guest-agent, and ssh.

  • Convert: Right-click the VM in Proxmox and select "Convert to Template".

Phase III: Physical Quorum

  • Cluster: Join Optiplex 7050 to the Optiplex 7020 cluster.

  • Arbitrator: Set up the Raspberry Pi as a headless qdevice to prevent "split-brain" scenarios.

Phase IV: Compute & Storage

  • RKE2: Spin up 3 VMs from your template (1 Master, 2 Workers) and run the basic RKE2 install script.

  • Longhorn: Deploy Longhorn using the default kubectl apply method to manage your replicated storage.

Phase V: Sentinel Command Center (Mattermost)

  • Placement: Deploy Mattermost on Server 2 (Optiplex 7050) as an LXC or VM.

  • Windows App: Install the Mattermost app on your Windows 11 PC.

  • Webhook: Create an "Incoming Webhook" so your server can send "Success" pings for later phases.

üö∂ Stage 2: The "Walk" (Observability)

Goal: Make the cluster "talk" by adding metrics and logs.

Phase VI: The Packaging Layer (Helm)

  • Setup: Install Helm on your DevOps node.

  • Strict Policy: From now on, only install apps using Helm to keep the system organized.

Phase VII: Unified Observability (Prometheus + Loki)

  • Metrics: Install kube-prometheus-stack via Helm.

  • Logs: Deploy Grafana Loki and Promtail to collect and view all system logs in one place.

  • Alerting: Connect Prometheus Alertmanager to Mattermost to receive Windows notifications.

Phase VIII: Uptime Kuma (Status Page)

  • Monitor: Deploy Uptime Kuma to track the "up/down" status of your services.

  • Notify: Link Kuma to Mattermost for instant status updates.

üèÉ Stage 3: The "Run" (Intelligence)

Goal: Full automation, high availability, and AI-driven interactive control.

Phase IX: GitOps Governance (ArgoCD)

  • Automation: Install ArgoCD and link it to your GitHub repository.

  • Sync: Move your configs to Git; ArgoCD will now automatically fix any "drift" in your cluster.

Phase X: High-Availability Messaging (RabbitMQ)

  • Deploy: Use ArgoCD to deploy a full HA RabbitMQ cluster.

  • Storage: Configure a 40Gi replicated volume in Longhorn with a 1Gi memory limit.

Phase XI: Intelligence Layer (Ollama RAG)

  • AI: Deploy Ollama on Server 2 (16GB RAM allocation).

  • RAG Pipeline: Feed your roadmaps and Loki logs into Ollama.

  • Matterchat: Interact with the AI via Mattermost to ask for cluster status summaries.

üîç Missing Information & Clarifications

  • Networking: Please confirm if your router supports Static DHCP/IP Reservation. If not, the plan must be adjusted to include manual Netplan configuration on every VM to prevent cluster failure.

  • Storage Space: With 500GB SSDs, ensure you have allocated roughly 100GB for the host OS and VM images before dedicating large chunks to Longhorn replicas.

  • Secrets: Before pushing to GitHub in Stage 3, you will need to choose a tool (like Sealed Secrets) to keep passwords safe.
